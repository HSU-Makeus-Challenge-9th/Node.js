너디너리 홈페이지 링크 : https://neordinary.co.kr/

1. 클라이언트는 `neordinary.co.kr` 서버와 통신하기 위해 먼저 TCP 3 way handshake 수행
   이 과정을 통해 양쪽 모두 데이터를 주고받을 준비 완료

2. 클라이언트는 요청 데이터와 서버의 IP 주소 및 PORT 번호가 담긴 패킷을 생성
   이 패킷은 클라이언트의 Access Network(예: 와이파이)를 통해 인터넷으로 보냄

3. 패킷이 인터넷의 Network Core에 도달하면, 여러 라우터를 거쳐 목적지인 서버를 향해 이동
   각 라우터는 패킷의 IP 주소를 보고 다음 경로를 결정하여 전달

4. 목적지(서버)에 도달하면, 패킷은 Physical Layer→ Data Link Layer → Network Layer 순으로 각 계층을 거칩니다.
    - Network Layer에서는 IP 주소를 확인해 올바른 수신지인지를 판단
    - Transport Layer에서는 PORT 번호를 확인해 해당 포트에서 실행 중인 애플리케이션으로 데이터를 전달

5. 서버는 응답이 담긴 새로운 패킷을 생성하여, 다시 인터넷 망의 여러 라우터를 통해 클라이언트에게 전달
   응답 패킷을 받은 클라이언트의 브라우저가 최종적으로 홈페이지 화면 보여주기
